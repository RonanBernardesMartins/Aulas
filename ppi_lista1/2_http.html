O que é protocolo HTTP

HTTP (Hypertext Transfer Protocol) é o protocolo de comunicação utilizado na World Wide Web para a transferência de dados entre clientes (navegadores) e servidores web.

Ele define como as mensagens são formatadas e transmitidas, bem como as ações que os servidores e navegadores devem tomar em resposta a várias solicitações.

  1. O cliente (navegador) envia uma solicitação HTTP ao servidor.(request)
  2. O servidor processa a solicitação e envia uma resposta HTTP de volta ao cliente.(response)

    Exemplo de uma requisição HTTP:

  1. Voce digita www.google.com no navegador.
  2. O navegador envia uma requisição HTTP GET ao servidor do Google.
  3. O servidor responde com um HTTP 200 OK e envia o conteúdo HTML da página solicitada.

Estrutura de uma requisição HTTP

Uma requisição HTTP é composta por três partes principais:

  1. Linha de requisição: contém o método HTTP, o caminho do recurso e a versão do protocolo.

    2. GET /index.html HTTP/1.1

  2. Headers: informações adicionais sobre a requisição, como tipo de conteúdo e autenticação.

    3. Host: www.exemplo.com
    User-Agent: Mozilla/5.0
    Accept: text/html

  3. Corpo (opcional): dados enviados ao servidor, geralmente em requisições POST.

Exemplo de uma requisição POST:

GET /produtos HTTP/1.1
Host: lojaonline.com.br
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
Accept: text/html,application/xhtml+xml
Accept-Language: pt-BR,pt;q=0.9
Connection: keep-alive

Explicando os campos:

GET → Método HTTP usado para solicitar dados do servidor.
/produtos → Caminho do recurso solicitado.
HTTP/1.1 → Versão do protocolo HTTP.
Host → Domínio do servidor que está sendo solicitado.
User-Agent → Identifica o navegador e o sistema operacional do cliente.
Accept → Tipos de conteúdo que o cliente pode processar.
Connection → Tipo de conexão desejada (manter viva ou fechar após a resposta).

Estrutura de uma Resposta HTTP (response)

A resposta HTTP do servidor contém:

  1. Linha de status:
    indica o resultado da solicitação (ex: HTTP/1.1 200 OK).
  2. Cabeçalhos:
    informações adicionais sobre a resposta, como tipo de conteúdo e comprimento.

    3. Content-Type: text/html; charset=UTF-8
    Content-Length: 2048
    Server: Apache/2.4.46

  3. Corpo (body):
    contém os dados da resposta, como o HTML da página solicitada.

Exemplo de Response:

HTTP/1.1 200 OK
Date: Tue, 04 Nov 2025 14:23:10 GMT
Server: Apache/2.4.46 (Ubuntu)
Content-Type: text/html; charset=UTF-8
Content-Length: 2480
Connection: keep-alive

Principais Métodos HTTP

Método Descrição
GET Solicita dados de um recurso.
POST Envia dados para o servidor.
PUT Atualiza um recurso existente.
DELETE Remove um recurso.
PATCH Aplica modificações parciais a um recurso.

Códigos de Status HTTP Comuns

Código Descrição Significado
200 OK Requisição bem-sucedida O servidor processou a requisição com sucesso
201 Created Criado O servidor criou um novo recurso
400 Bad Request Erro do cliente Requisição inválida
401 Unauthorized Não autorizado Necessita autenticação
404 Not Found Não encontrado O recurso solicitado não foi encontrado
500 Internal Server Error Erro interno do servidor Ocorreu um erro inesperado no servidor
